1.vue spa 是下载完js然后执行，再渲染出dom
如果网速很慢，js下载耗时会变长，此时用户界面会一直停留在白屏阶段
直到js下载完毕后，再在客户端组装成html

2.vue ssr 是本地服务器使用本地js进行组装成html，
然后客户端直接渲染收到的html，最后再去下载js进行客户端混合

3.使用vue ssr时 使用router-link或nuxt-child的to属性进行导航
会走单页应用渲染，所以要使用a标签的href或:href进行导航，
或者使用window.location.href进行导航

4.如果某个页面渲染时要下载的js体积较小，直接使用单页渲染体验会更好
--用户界面不会出现闪屏的情况，而且爬虫也能在服务端爬到该页（因为服务端也可以渲染）

* 首次进入应用时，应用要下载的第三方js文件较大，故本项目只做首页服务端渲染，其他页做单页渲染（不用担心seo的问题，因为服务端也能渲染）*

5.asyncData - 单页应用渲染时会在客户端自动调用，
              走服务端渲染时会在服务端自动调用
  #前提是仅限nuxt中...

"config": {
    "nuxt": {
        "host": "0.0.0.0", // 允许所有主机ip访问

    }
